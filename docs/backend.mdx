## 2. Chat 端点详解

### POST /sessions/{id}/chat

文件: `backend/app/api/v1/agents.py`

```python
class ChatRequest(BaseModel):
    message: str = Field(..., min_length=1, max_length=2000)
    patient_history: str | None = Field(None, max_length=5000)  # [TODO] 前端未使用
```

**守卫**:
1. Session 存在性检查 (404)
2. `context.interview.phase == "completed"` 检查 (400 + `[CHAT-400]` 日志)
3. 空响应检测 → `FAILED` 状态 + error 事件

**LLM 调用**: `llm.chat()` 非流式 + `_chunk_text()` SSE 分块。注：原使用 `llm.chat_stream()` 流式，但 kimi-k2.6 流式 API 挂死，改为非流式。

**`_build_chat_context` 上下文构建**:
- `parent.structured_output` — 诊断摘要
- `parent.context.lab_reports` — 检查报告指标
- `parent.context.interview.collected_info` — 病史
- sibling CONVERSATION sessions — 历史对话
- historical MedicalCase — 注册用户历史病例

## 5. 化验单 Bridge 归一化

`backend/app/api/v1/agents.py` `store_lab_reports`:

```python
# Bridge key 归一化: UUID → _frontend_sid
bridge_key = session_id
try:
    sid = uuid.UUID(session_id)
    _s = await db.get(AgentSession, sid)
    fsid = (_s.context or {}).get("_frontend_sid")
    if fsid:
        bridge_key = fsid  # 统一用前端 ID 做 key
except ValueError:
    pass

prev = _session_lab_bridge.get(bridge_key, [])
prev.extend(reports)
_session_lab_bridge[bridge_key] = prev

await _update_interview_session_lab_data(db, session_id, prev)
```

`_update_interview_session_lab_data` 双匹配策略:
- Strategy 1: UUID PK 直接匹配
- Strategy 2: `_frontend_sid` 匹配（不限 status）

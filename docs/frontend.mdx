## 2. 核心状态：chatMode

```typescript
type ChatMode = 'idle' | 'consulting' | 'diagnosed';

// idle:       初始态，显示快捷回复
// consulting: 问诊中，显示 PendingCardsPanel
// diagnosed:  诊断完成，显示 UploadStatusBanner + ChatInput
```

**状态转换**:
```
idle → (发送主诉) → consulting
consulting → (structured 事件) → diagnosed
diagnosed → (startNewSession) → idle
```

## 3. 上传流程

### handleFileUpload (ChatPage.tsx)

```
用户选择文件
  → uploadDocument() 上传到服务器
  → 消息流插入 "正在解析..." (UploadReportCard parsing)
  → activeUploads 追踪 (全模式)
  → setInterval 轮询 getParseResult() (每 1.5 秒)
       ├─ completed → setActiveUploads + store_lab_reports POST
       └─ failed    → setActiveUploads + failedFileAttempts++
```

### store_lab_reports 的 session ID

```typescript
const uploadSessionId = backendSessionIdRef.current || currentSessionId;
```

### UploadStatusBanner 显示条件

```typescript
activeUploads.length > 0 || (chatMode === 'diagnosed' && !uploadBannerDismissed.current)
```

### UploadReportCard 三态

| 状态 | 视觉 | 交互 |
|------|------|------|
| 解析中 (parsing) | 暖金渐变 + 🔄 脉冲动画 | 纯展示 |
| 已完成 (completed) | 浅绿渐变 + ✅ + LabReportCard 默认展开 | 纯展示 |
| 失败 (failed) | 暖橙渐变 + ❌ + 错误信息 | 纯展示 |

## 4. SSE 事件处理

### streamDiagnoseContinue (interview 路径)

文件 `api/agent.ts` `streamDiagnoseContinue()`:
- 处理: `thinking`, `tool_call`, `tool_result`, `text`, `structured`, `question`, `complete`, `error`
- `structured` → setChatMode('diagnosed'), setReportData()
- `complete` → set backendSessionIdRef.current

### streamChat (chat 路径)

文件 `api/agent.ts` `streamChat()`:
- POST + fetch + ReadableStream SSE 解析
- 处理: `text`, `complete`, `error`

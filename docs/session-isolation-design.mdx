# 会话隔离与自动停止旧对话设计方案

> 最后更新: 2026-06-06

## 1. 核心设计目标

- **会话隔离**：每个对话拥有独立上下文
- **自动停止旧对话**：用户开启新对话时，前端主动断开旧对话的流式连接，后端立即终止对应的推理任务
- **资源生命周期管理**：包括计算资源、内存/缓存、数据库连接等
- **前端体验**：无延迟切换，界面不再显示旧对话输出

## 2. 整体架构

```
[前端] <--SSE--> [API Gateway] --> [推理服务(LLM)]
                    |
                    +--> [会话管理器(Redis)]
                    +--> [数据库(对话历史)]
```

## 3. 关键数据结构

### 3.1 会话对象（Redis）

```json
{
  "session_id": "uuid",
  "user_id": "user_123",
  "status": "active | stopped | expired",
  "inference_task_id": "task_456",
  "created_at": 1718000000,
  "last_active_at": 1718000100,
  "ttl": 1800
}
```

## 4. 交互流程

### 4.1 开启新对话
1. 前端调用 `POST /api/v1/chat/session` 创建新会话
2. 后端生成 session_id，存入 Redis，返回给前端
3. 前端切换激活的 session_id

### 4.2 开启新对话时停掉旧对话
1. 前端获取旧会话 ID（old_session_id）
2. 前端调用 `POST /api/v1/chat/abort`，携带 old_session_id（异步）
3. 前端关闭旧 SSE 连接
4. 前端创建新会话，切换本地激活 ID
5. 后端收到 /abort：
   - 从 Redis 读取旧会话
   - 向推理服务发送取消请求
   - 将 status 改为 stopped

### 4.3 会话闲置自动清理
定时任务扫描 Redis 中 last_active_at 超过 TTL 的会话，清理资源。

## 5. 前端实现要点

```javascript
let activeSessionId = null;

async function newConversation() {
  if (activeSessionId) {
    fetch('/api/v1/chat/abort', {
      method: 'POST',
      body: JSON.stringify({ session_id: activeSessionId })
    }).catch(console.error);
    if (currentEventSource) {
      currentEventSource.close();
      currentEventSource = null;
    }
  }
  const res = await fetch('/api/v1/chat/session', { method: 'POST' });
  const { session_id } = await res.json();
  activeSessionId = session_id;
  clearChatHistory();
}
```

## 6. 资源清理与异常处理

| 场景 | 处理方式 |
|------|----------|
| 用户关闭浏览器 | beforeunload 事件调用 /abort |
| 网络断开 | SSE 写失败 → 标记会话为 stopped |
| 推理服务崩溃 | 超时未响应则清理 |
| 并发限制 | 每用户最多 N 个活跃会话 |

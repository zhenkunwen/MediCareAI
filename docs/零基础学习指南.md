#### 📌 async / await（异步编程）

这是项目**最重要**的概念，因为所有 LLM 调用都是异步的。

```python
import asyncio

# 异步函数——定义时加 async
async def fetch_data() -> str:
    """模拟从网络获取数据"""
    await asyncio.sleep(2)   # 模拟等待 2 秒
    return "数据加载完成"

# 调用异步函数
async def main():
    print("开始获取...")
    result = await fetch_data()  # await 等待结果，这 2 秒内可以做其他事
    print(result)

# 运行
asyncio.run(main())
```

> **🔑 关键理解**：`async` 定义异步函数，`await` 等待异步结果。
> 就好像你点了一杯咖啡（异步任务），拿到一个号码牌（await），咖啡做好了叫你去拿——这中间你可以刷手机（做其他事）。

## 3. 阶段二：FastAPI——看懂接口

> ⏱ 预计：1 周

### 3.1 FastAPI 是干什么的

FastAPI 是一个 Python Web 框架，负责：
1. **接收 HTTP 请求**（用户浏览器发来的）
2. **处理请求**（调用后端逻辑）
3. **返回响应**（JSON 数据给前端）

### 3.2 路由定义

项目里路由在 `backend/app/api/v1/` 下，每个文件管理一类接口。

```python
from fastapi import APIRouter, Depends

# 创建一个路由器（掌管 /api/v1/users 下的所有接口）
router = APIRouter(prefix="/api/v1/users", tags=["用户管理"])

# 定义 GET 请求
@router.get("/")          # 访问 GET /api/v1/users/
async def list_users():
    return {"users": []}

# 定义 POST 请求（创建资源）
@router.post("/")         # 访问 POST /api/v1/users/
async def create_user(data: UserCreate):  # data 自动校验
    return {"id": 1, **data.model_dump()}

# 带路径参数的 GET
@router.get("/{user_id}") # 访问 GET /api/v1/users/123
async def get_user(user_id: int):
    return {"user_id": user_id}
```

> **🔑 一句话理解**：`@router.get("/")` 就是"当用户 GET 访问这个路径时，执行下面的函数"。

### 3.3 依赖注入 Depends

这是 FastAPI 最强大的功能。

```python
# 定义一个"依赖"——获取当前登录用户
async def get_current_user(token: str = Header(...)) -> User:
    """从请求头中解析 token，返回用户对象"""
    user = await verify_token(token)
    return user

# 在路由中使用这个依赖
@router.get("/profile")
async def get_profile(
    current_user: User = Depends(get_current_user)  # FastAPI 自动调用
):
    # 到这里 current_user 已经是登录用户了
    return {"email": current_user.email, "name": current_user.full_name}
```

> **🔑 理解**：`Depends` 就像"在调用路由函数前，先帮我做一件事"。项目里用它来做：
> - 用户认证（检查是否登录）
> - 角色检查（是患者还是医生）
> - 数据库会话管理

### 4.1 数据模型 vs 数据库表

```python
# backend/app/models/user.py
from sqlalchemy import Column, String, Integer, Boolean, DateTime
from sqlalchemy.orm import DeclarativeBase

class Base(DeclarativeBase):
    pass

class User(Base):
    """用户表——一个类对应一张数据库表"""
    __tablename__ = "users"     # 表名
    
    id = Column(Integer, primary_key=True)           # 主键，自增
    email = Column(String(255), unique=True)         # 唯一约束
    hashed_password = Column(String(255))            # 加密后的密码
    full_name = Column(String(255))
    role = Column(String(20), default="patient")     # 角色：patient/doctor/admin
    is_active = Column(Boolean, default=True)
    created_at = Column(DateTime)                    # 创建时间
```

> 每一行 = 一个用户，每一列 = 一个属性。和 Excel 表格一模一样。

### 4.2 增删改查（CRUD）

```python
# 查询用户
async def get_user_by_email(db: AsyncSession, email: str) -> User | None:
    """
    根据邮箱查询用户
    db: 数据库会话（类似 Excel 文件句柄）
    """
    result = await db.execute(
        select(User).where(User.email == email)
    )
    return result.scalar_one_or_none()  # 返回一个用户或 None

# 创建用户
async def create_user(db: AsyncSession, data: dict) -> User:
    user = User(**data)         # 创建 User 对象
    db.add(user)                # 添加到会话
    await db.commit()           # 提交事务（保存到数据库）
    await db.refresh(user)      # 刷新（获取数据库自动生成的 id）
    return user

# 更新用户
async def update_user(db: AsyncSession, user_id: int, data: dict) -> User:
    user = await db.get(User, user_id)
    for key, value in data.items():
        setattr(user, key, value)  # 逐个更新字段
    await db.commit()
    return user

# 删除用户
async def delete_user(db: AsyncSession, user_id: int) -> None:
    user = await db.get(User, user_id)
    await db.delete(user)
    await db.commit()
```

### 5.2 五个核心文件逐个拆解

#### ① `services/llm.py` — LLM 调用封装（最重要的文件）

这个文件是所有 AI 能力的入口，负责**调用大模型**。

```python
# 简化后的核心逻辑
class LLMService:
    """
    统一 LLM 服务
    职责：接收消息 → 调 AI → 返回结果
    """
    
    async def chat(
        self,
        messages: list[dict],    # 对话历史
        system_prompt: str = "", # 系统提示词（设定 AI 角色）
        model: str | None = None # 模型名，None 则用默认
    ) -> LLMResponse:
        """
        最常用的方法：普通对话
        """
        # 1. 检查 token 预算（防止滥用）
        await self._check_budget(user_id)
        
        # 2. 检查语义缓存（相同问题直接返回，省钱）
        cached = await self._check_cache(messages, system_prompt)
        if cached:
            return cached
        
        # 3. 调用 OpenAI API
        response = await self.client.chat.completions.create(
            model=model or self.default_model,
            messages=[{"role": "system", "content": system_prompt}] + messages
        )
        
        # 4. 记录 token 消耗
        await self._deduct_budget(user_id, response.usage.total_tokens)
        
        return LLMResponse(
            content=response.choices[0].message.content,
            token_usage=response.usage.total_tokens
        )
```

> **🔑 你只需要知道**：这个文件就是"调 AI 的地方"。项目里所有 Agent 最终都是调这个文件。

#### ② `tools/base.py` + `tools/registry.py` — 工具系统

AI 需要能查数据库、搜知识库，这就是"工具"。

```python
# tools/base.py — 工具基类
class Tool:
    """所有工具的父类"""
    
    name: str           # 工具名（如 "query_patient_history"）
    description: str    # 描述：告诉 AI 这个工具是干什么的
    
    def to_openai_schema(self) -> dict:
        """转成 OpenAI 能理解的工具格式"""
        return {
            "type": "function",
            "function": {
                "name": self.name,
                "description": self.description,
                "parameters": {...}  # 参数定义
            }
        }
    
    async def execute(self, **kwargs) -> str:
        """执行工具——子类必须实现"""
        raise NotImplementedError

# tools/registry.py — 工具注册表
class ToolRegistry:
    """
    所有工具的"电话本"
    - 注册：告诉系统有哪些工具可用
    - 执行：根据工具名找到对应的工具并运行
    """
    
    def __init__(self):
        self._tools: dict[str, Tool] = {}
    
    def register(self, tool: Tool):
        """注册工具"""
        self._tools[tool.name] = tool
    
    def list_schemas(self) -> list[dict]:
        """列出所有工具的 OpenAI 格式定义"""
        return [t.to_openai_schema() for t in self._tools.values()]
    
    async def execute(self, name: str, **kwargs) -> str:
        """根据名字找到工具并执行"""
        tool = self._tools.get(name)
        if not tool:
            raise ValueError(f"工具 {name} 不存在")
        return await tool.execute(**kwargs)
```

> **🔑 理解**：AI 不能直接操作数据库，所以给它"工具"。AI 说"我想查病人历史"，系统就调 `query_patient_history` 工具，查到结果返回给 AI。

#### ④ `services/orchestrator.py` — 问诊编排器

这是整个项目中**最核心的决策逻辑**。

```python
class InterviewOrchestrator:
    """
    问诊编排器——项目的"大脑"
    决定：该问什么问题？问够了没有？是不是该出诊断了？
    """
    
    async def decide_next(
        self, 
        interview_state: InterviewState
    ) -> NextAction:
        """
        【核心决策函数】
        每次用户回答完问题后，系统调用这个函数决定下一步做什么
        
        返回：
        - ask_question: 继续问问题
        - prepare_synthesis: 信息够了，准备生成诊断
        - request_lab: 建议做检查
        """
        
        # 1. 检查是否所有维度都收集了
        if not interview_state.all_dimensions_collected():
            # 还有临床维度没问到，继续提问
            next_question = await self._generate_question(interview_state)
            return NextAction.ask_question(next_question)
        
        # 2. 检查用户是否有待回答的问题
        if interview_state.has_pending_questions():
            # 用户还有问题没回答，等回答
            return NextAction.wait_for_answer()
        
        # 3. 检查是否需要化验
        if interview_state.needs_lab_test() and not interview_state.has_lab():
            return NextAction.request_lab()
        
        # 4. 所有条件满足，生成诊断
        return NextAction.prepare_synthesis()
```

> **🔑 一句话理解**：`decide_next()` 就像医生问诊时的内心独白——"症状问完了吗？检查做完了吗？好了，可以下诊断了"。

#### ⑤ `services/rag.py` — RAG 检索

RAG = Retrieval-Augmented Generation（检索增强生成）

```python
class RAGService:
    """
    检索增强生成
    作用：从知识库中找到相关文档，帮助 AI 更准确回答
    
    为什么需要 RAG？
    医学知识不断更新，AI 的训练数据可能过时。
    RAG 让 AI 先查最新的医学资料，再回答问题。
    """
    
    async def retrieve(
        self, 
        query: str,           # 用户的查询
        top_k: int = 5        # 返回几条相关文档
    ) -> list[Document]:
        """
        检索流程：
        1. 把用户的查询转成"向量"（数学上的坐标）
        2. 在知识库中找"最接近"的文档
        3. 返回匹配的文档
        """
        # 1. 生成查询的向量
        query_embedding = await self.embedding_service.embed(query)
        
        # 2. 向量相似度搜索
        results = await self.vector_store.similarity_search(
            query_embedding, top_k
        )
        
        # 3. 重排序（让最相关的结果排前面）
        reranked = await self.reranker.rerank(query, results)
        
        return reranked
```

## 8. 术语表

| 术语 | 中文 | 通俗解释 |
|------|------|----------|
| **API** | 接口 | 程序之间的"对话窗口" |
| **Endpoint** | 端点 | 一个具体的 API 地址，如 `/api/v1/auth/login` |
| **REST** | 表现层状态转换 | 用 GET/POST/PUT/DELETE 操作资源的风格 |
| **SSE** | 服务器推送事件 | 服务器"源源不断"向前端推送数据（如 AI 逐字输出） |
| **ORM** | 对象关系映射 | 用 Python 类操作数据库表，不用写 SQL |
| **CRUD** | 增删改查 | Create/Read/Update/Delete |
| **RAG** | 检索增强生成 | 先查知识库，再让 AI 回答 |
| **Agent** | 智能体 | 能独立完成特定任务的 AI 程序 |
| **LLM** | 大语言模型 | 如 GPT-4、DeepSeek 等 AI 模型 |
| **Prompt** | 提示词 | 给 AI 的指令 |
| **Tool Call** | 工具调用 | AI 请求调用某个函数（如查数据库） |
| **Schema** | 模式/结构定义 | 数据的格式定义 |
| **Dependency Injection** | 依赖注入 | FastAPI 自动帮你准备好需要的对象 |
| **Middleware** | 中间件 | 请求到达路由前/后执行的代码 |
| **Context Manager** | 上下文管理器 | `with` / `async with` 语句，自动管理资源 |
| **Vector** | 向量 | 用一串数字表示"语义"，用于相似度搜索 |
| **Embedding** | 嵌入 | 把文本转成向量的过程 |
| **JWT** | JSON Web Token | 用户登录后的"身份证" |
| **Decorator** | 装饰器 | Python 的 `@` 语法，给函数"加功能" |
